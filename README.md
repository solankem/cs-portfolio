# cs-300

What was the problem you were solving in the projects for this course?

This course covered various data models (Binary trees, Hash Tables, Linked Lists) and their suitable applications within software.

How did you approach the problem? Consider why data structures are important to understand.

I made an effort to grasp the relative strengths and weaknesses of every data structure and utilize them when their strengths 
outweighed their weaknesses.

How did you overcome any roadblocks you encountered while going through the activities or project?

I addressed any challenges by consulting the official documentation of the method or searching for example applications. 
Alternatively, at times, I created a small piece of prototype code that was straightforward and then built upon it as the logic grew 
more intricate.

How has your work on this project expanded your approach to designing software and developing programs?

The big-O analysis prompted me to reflect on performance and my strategy for algorithmic design. It also revealed that my thinking 
does not often employ recursion, which is a limitation in some of my possible designs.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

I believe all these components stem from experience. I've created enough file streams that I now have a standard function I use as a 
template. I may even take some of the other components I frequently reuse and turn them into libraries.
